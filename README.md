# Gitea OpenCode

Gitea OpenCode is a Dockerized adapter that runs OpenCode from Gitea Actions. It lets users trigger code review, issue explanation, issue fixes, and PR changes with comments such as `/opencode` or `/oc`.

## Features

- `/opencode explain this issue` on an Issue replies with an explanation.
- `/opencode fix this` on an Issue creates a branch, commits changes, pushes it, and opens a PR.
- `/oc add error handling here` on a PR comment or review line comment commits changes to the same PR branch.
- PR `opened`, `reopened`, and `synchronized` events run an automatic review.
- `workflow_dispatch` runs a manual OpenCode prompt and can create a PR when files change.

## Build The OpenCode Image

Build and push the image used by the Gitea runner label:

```sh
docker build -t ghcr.io/your-org/gitea-opencode:latest .
docker push ghcr.io/your-org/gitea-opencode:latest
```

The image contains Node 22, Git, curl, jq, and the `opencode-ai` CLI.

## Configure A Gitea Runner

Gitea Actions must be enabled and a runner must be registered. The workflow generated by this project defaults to `runs-on: opencode`, so configure a runner label like:

```text
opencode:docker://ghcr.io/your-org/gitea-opencode:latest
```

You can adapt `docker-compose.runner.yml` and `runner-config.example.yaml`:

```sh
export GITEA_INSTANCE_URL="https://gitea.example.com"
export GITEA_RUNNER_REGISTRATION_TOKEN="..."
export GITEA_RUNNER_LABELS="opencode:docker://ghcr.io/your-org/gitea-opencode:latest"
docker compose -f docker-compose.runner.yml up -d
```

Mounting `/var/run/docker.sock` gives the runner access to the host Docker daemon. Use only trusted runners for trusted repositories.

## Install In A Repository

From a target Git repository:

```sh
path/to/gitea-opencode/install-opencode.sh
```

Windows PowerShell:

```powershell
path\to\gitea-opencode\install-opencode.ps1
```

Useful options:

```sh
./install-opencode.sh --dry-run
./install-opencode.sh --force --no-push
./install-opencode.sh --runner-label opencode --model anthropic/claude-sonnet-4-20250514
```

The installer writes `.gitea/workflows/opencode.yml`, commits it with `chore: add gitea opencode workflow`, and pushes the current branch unless `--no-commit` or `--no-push` is used.

## Required Secrets

Set these repository or organization Actions secrets:

- `ANTHROPIC_API_KEY`: required for the default Anthropic model.
- `OPENCODE_GITEA_TOKEN`: optional PAT override when the built-in `${{ secrets.GITEA_TOKEN }}` is too restricted.

The workflow requests write permissions for code, issues, and pull requests. Gitea may clamp these permissions in repository or owner Actions settings.

## Supported Commands

Issue commands:

```text
/opencode explain this issue
/opencode fix this
/oc fix this bug
```

PR commands:

```text
Delete the attachment from S3 when the note is removed /oc
/oc add error handling here
```

PR line comments are supported when Gitea includes file, line, and diff context in the review comment payload.

## Schedule Template

Schedule is intentionally not enabled by default because it has no Issue or PR context. To add a scheduled review, add this trigger and a fixed prompt to the generated workflow:

```yaml
on:
  schedule:
    - cron: "0 9 * * 1"

env:
  OPENCODE_PROMPT: "Review TODO comments and summarize important follow-ups."
```
